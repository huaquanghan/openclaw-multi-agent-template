# USAGE.md - How to Use Tí Cận

## When to Call Tí Cận

### Perfect For
- Writing code
- Debugging issues
- Building tools and scripts
- Code review
- Technical architecture
- Automation
- API integrations

### Not Ideal For
- Visual design (delegate to Tiểu Hoa)
- Research (delegate to Mắt Cú)
- Scheduling (delegate to Tiểu Mỉ)
- Strategic decisions (escalate to Cọp)

## How to Delegate

### Feature Development

```
Tí Cận, build [feature].

Requirements:
- [Requirement 1]
- [Requirement 2]

Context: [Background]
Tech stack: [Languages/frameworks]
Deadline: [When needed]
```

### Bug Fix

```
Tí Cận, fix this bug:
**Issue:** [Description]
**Error:** [Error message]
**Steps to reproduce:**
1. [Step 1]
2. [Step 2]

**Expected:** [What should happen]
**Actual:** [What happens]
```

### Code Review

```
Tí Cận, review this code:
[Code or file path]

Focus on: [Specific aspects]
```

## Example Requests

### Build a Tool
```
Tí Cận, create a Python script that:
- Reads CSV files from a directory
- Validates the data
- Generates a summary report
- Saves results to a new CSV
```

### Debug an Issue
```
Tí Cận, help debug this error:
[Error message/stack trace]

The code is:
[Relevant code]
```

### Create a Web Page
```
Tí Cận, build a landing page with:
- Hero section with title and CTA
- Features section (3 cards)
- Contact form
- Responsive design
```

## Expected Output

Tí Cận will provide:
- Working code
- Explanation of implementation
- Testing notes
- Any setup instructions
- Documentation if needed

## Tips for Best Results

1. **Provide context** - What is this for? Who will use it?
2. **Specify constraints** - Tech stack, performance requirements, etc.
3. **Share examples** - "Like X but with Y"
4. **Define done** - What does success look like?

## Code Standards

Tí Cận follows:
- Clean code principles
- Proper error handling
- Comments for complex logic
- Consistent formatting

## Response Time

- **Quick fixes:** 2-5 minutes
- **Small features:** 10-20 minutes
- **Complex features:** 30+ minutes (with progress updates)

## Handoff Checklist

When Tí Cận completes work:
- [ ] Code is tested
- [ ] Documentation provided
- [ ] Setup instructions included
- [ ] Known issues documented
